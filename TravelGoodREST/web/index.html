REST api description

Description of itineraries resource

-> Invoke GET on itineraries resource to get a list of itineraries
Not part of original requirements
->Invoke POST on itineraries to instantiate a new itinerary. Returns Itinerary object and resource location
->Invoke GET on itineraries/{itineraryID} to get the Itinerary object with the same ID
->Invoke PUT on itineraries/{itineraryID}/creditcard with an object of type CreditCardInfo to set creditcard for itinerary({
-> Invoke PUT on itineraries/{itineraryID}/flights with a FlightInformation object to add a flight to the itinerary
-> Invoke PUT on itineraries/{itineraryID}/hotels with a HotelInformation object to add a hotel to the itinerary
->Invoke PUT on itinerarues/{itineraryID}/state with a String "PAID" to pay for Itinerary or "CANCELED" to cancel itinerary

Description of flights resource

->Invoke GET on flights with parameters: String origin, String origin, String date on the form "yyyy-mm-dd" to get a list of FlightInformation

Description of flights resource

->Invoke GET on hotels with parameters: String city, String arrival, String departure on the form "yyyy-mm-dd" to get a list of FlightInformation
